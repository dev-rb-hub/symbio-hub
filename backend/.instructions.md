# Backend Agent Instructions

This file helps AI agents working in the `backend` workspace.

## Focus
- Keep backend work inside `backend/` and avoid introducing frontend dependencies.
- Preserve the existing backend architecture and interfaces.
- Prefer changes in the API and infrastructure layers rather than cross-cutting frontend behavior.

## Common tasks
- Use `dotnet test` to validate backend changes.
- Use `dotnet run --project backend/Symbio.API` when running the API locally.
- Follow existing repository conventions for branch names and commit messages.

## Agent guidance
- Do not leak backend database structures or internal API implementation details into frontend code.
- Use the root repo guidance in `AGENTS.md` for broader project conventions.
- Keep backend implementation changes minimal and aligned with the current `.NET` architecture.
